## Why Have a Blog?

其實要分成什麼時期的我來看待這件事，如果是在 3 年前的我會說因為研究所上有很多人都用 [Jekyll](https://jekyllrb.com/) 或 [Hugo](https://gohugo.io/) 等技術來生成靜態網頁架在 github pages 上。當時我也嘗試這麼做，但效果不佳，因為~懶惰~覺得麻煩所以就草草亂架。架好的網也站不盡人意，要馬覺得色調太花，要馬覺得無法好好的客製化自己想要的東西，例如 search bar。基於這些因素，再加上日後我開始參與所上教授們的多項計畫，就一直拖到現在了。

## Three Years Later

現在的我突然想要建 blog 有兩大主因，一個是最近因要面試開始大量閱讀一些與工程相關的知識，想要強迫自己記下來以便能更好地理解，另一個則是在當了 3 年的全端工程師，學到了如何用 [NextJs](https://nextjs.org/) 來刻出一個網站，且在客製化網站上的能力也比以前更好。

總之，我就用 NextJS 搭配 [Mantine UI](https://mantine.dev/) 在一天的時間內做出一個大概的 blog 網站，然後部署在 github pages 上。但這不會是長久之計，因為 github pages 是有容量限制的[^1]，購買屬於自己的網域是勢在必行。

## Why NextJS?

之所以採用容量相對較大的 NextJS，主要是因為他的 SSG (Static Site Generation) 和 SSR (Server Side Rendering) 實在是太強大了，讓我在渲染 [mdx](https://mdxjs.com/)[^2] 時快如閃電。除了上述，像是檔案架構即 routing 等優點，都是我日後選擇它來開發前端頁面的主因。

[^1]: Github Pages 在幾年前開始限制空間與流量，文件大小上限 1G，而流量則為 100 G。
[^2]: mdx 為 markdown 的擴充格式，它會被編譯成 JSX element，非常適合 React 的 SSG。

## 展望

其實我在刻前端時，主要是參考一個我很常看的程式部落客 - [Kenny's Blog](https://kennychen-blog.herokuapp.com/)，然後用 NextJs 和 Mantine UI 去一一實現一些我覺得蠻不錯的功能，像是 [archives](/blog/archives?page=1) 顯示了歷年的文章 ~，也可以順便讓我看看自己的熱度到底會有幾分鐘~。寫這篇時的網站應該還很陽春，很多功能我都還沒有補齊，例如顯示 tags 的頁面等等，希望之後有時間可以補上。
