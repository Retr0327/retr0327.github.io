理解 GROUP BY

發表於 2023-04-16

sql

GROUP BY

GROUP BY 是一種在 SQL 中常用的功能，它允許我們按照一定的規則將數據進行分組。假設我們有一個 engineers table，其結構如下：

idnamelanguage
1AlicePython
2BobGo
3CharlieC
4DavePython
5EveTypeScript
6FrankTypeScript
7GraceJava
8HeidiPython

假設我們想要統計「每個語言的使用人數」，我們可以用 GROUP BY 來達成。以下例子我們以 PostgreSQL 為例：

SELECT language, COUNT(name) AS "numbers"
FROM engineers
GROUP BY language;

這個 SQL query 先將工程師按 language 分組，然後對每個分組應用 COUNT 函數，計算每個分組的數量。因此，我們會得到以下結果：

languagenumbers
Python3
Go1
C1
TypeScript2
Java1

我們可以通過這種方式對數據進行分組統計，以便更好地了解數據分佈和分類情況。

GROUP BY 原理分析

當 GROUP BY 在執行數據分組時，會使用一個 temporary table (臨時表) 來存儲每個分組的數據。分組後，我們可以對每組數據進行各種聚合（aggregate）操作，例如 SUM、AVG、COUNT 等。這個過程可以看作是「化整為零策略（split-apply-combine）」1的一個具體實現，流程如下：

  • Split：根據 FROM \<table-name\> GROUP BY \<column-name\>，將每個分組的數據存儲到臨時表中。2
  • Apply：根據 SELECT \<column-name\>，對每個分組的數據進行聚合操作。
  • Combine：將聚合結果返回。

以下我們以 engineers table 為例，來了解 GROUP BY 的具體運行原理。

Split

FROM engineers GROUP BY language 在被執行後，生成了一個 temporary table，如下所示：

idnamelanguage
1
4
8		Alice
Dave
Heidi		Python
2BobGo
3CharlieC
5
6		Eve
Frank		TypeScript
7GraceJava

GROUP BY language 會先找到 language 這個欄位，然後將具有「相同 language」的值分組，並合併成一行（將所有 id 和 name 寫到一個單元格中）。

Apply

在第二階段會對每個子及應用一個「聚合函數 (aggregate functions)」，然後將結果組合起來。但不知道你有沒以想過為何要用聚合函數 (aggregate functions)？

原因在於像 PostgreSQL 這樣的 RDBMS (Relational Database Management System) 是基於「關係」的，也就是一個單元格位「不允許有多個值」。舉例來說，如果我們執行 SELECT * FROM engineers GROUP BY language 會得到 temporary table 的結果：

idnamelanguage
1
4
8		Alice
Dave
Heidi		Python
2BobGo
3CharlieC
5
6		Eve
Frank		TypeScript
7GraceJava

但由於 RDBMS，不允許一個單元格有多個值，執行 SELECT * 後就會報錯。因此，我們必須將每個分組的數據進行聚合，因為聚合函數 (aggregate functions) 的功能就是用來輸入多個數據並輸出一個數據的。

Combine

選定好要使用哪種聚合函數 (aggregate functions) 後，我們就可以將聚合結果組合起來，並返回結果。在我們之前的例子中，我們使用 COUNT 函數，來計算每個 language 的使用人數，因此得到以下結果：

languagenumbers
Python3
Go1
C1
TypeScript2
Java1

Footnotes

  1. Split-apply-combine 指的是將數據集拆分成多個子集，對每個子集應用一個函數，然後將結果組合起來。在 SQL 中，GROUP BY 拆分數據集，並用聚合函數（aggregate functions）對每個子集進行計算，然後將結果合併起來。 ↩
